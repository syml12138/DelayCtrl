# DelayCtrl — 四路继电器 UDP 控制上位机

一个基于 **WinForms (.NET 10)** 的四路继电器控制上位机程序，通过 **UDP 协议** 向继电器设备发送开关指令，并实时显示设备返回的状态。

![.NET 10](https://img.shields.io/badge/.NET-10.0-blue)
![Platform](https://img.shields.io/badge/Platform-Windows-lightgrey)
![License](https://img.shields.io/badge/License-MIT-green)

---

## 功能特性

- ✅ **UDP 通信**：通过 UDP 协议与继电器设备通信，支持自定义 IP 地址和端口
- ✅ **单路控制**：独立控制第 1 ~ 4 路继电器的开启/关闭
- ✅ **全通道控制**：一键开启/关闭全部 4 路继电器
- ✅ **实时状态反馈**：解析设备返回的状态字节，界面指示灯实时更新（绿色=开启，灰色=关闭）
- ✅ **通信日志**：底部日志区以十六进制格式记录所有收发数据，便于调试

## 通信协议

### 连接方式

| 参数     | 说明                           |
| -------- | ------------------------------ |
| 协议     | UDP                            |
| 默认 IP  | `192.168.1.100`（需替换为实际设备地址） |
| 默认端口 | `8080`（需替换为实际设备端口）         |

### 指令格式

**发送指令**（上位机 → 设备）：3 字节

```
[0x00] [命令字节] [0xFF]
```

**设备响应**（设备 → 上位机）：4 字节

```
[0x00] [命令字节] [0xFF] [0xEF]
```

### 指令对照表

| 操作       | 发送指令（HEX） | 设备响应（HEX）  |
| ---------- | --------------- | ---------------- |
| 第一路 开启 | `00 F1 FF`      | `00 F1 FF EF`   |
| 第一路 关闭 | `00 01 FF`      | `00 01 FF EF`   |
| 第二路 开启 | `00 F2 FF`      | `00 F2 FF EF`   |
| 第二路 关闭 | `00 02 FF`      | `00 02 FF EF`   |
| 第三路 开启 | `00 F3 FF`      | `00 F3 FF EF`   |
| 第三路 关闭 | `00 03 FF`      | `00 03 FF EF`   |
| 第四路 开启 | `00 F4 FF`      | `00 F4 FF EF`   |
| 第四路 关闭 | `00 04 FF`      | `00 04 FF EF`   |
| 全通道 开启 | `00 F9 FF`      | `00 F9 FF EF`   |
| 全通道 关闭 | `00 09 FF`      | `00 09 FF EF`   |

### 命令字节规律

| 操作 | 计算规则             | 示例                          |
| ---- | -------------------- | ----------------------------- |
| 开启 | `0xF0 + 通道号(1~4)` | 第 2 路开 = `0xF0 + 2 = 0xF2` |
| 关闭 | `0x00 + 通道号(1~4)` | 第 3 路关 = `0x00 + 3 = 0x03` |
| 全开 | 固定 `0xF9`          | —                             |
| 全关 | 固定 `0x09`          | —                             |

## 快速开始

### 环境要求

- **操作系统**：Windows 10 / 11
- **运行时**：[.NET 10 SDK](https://dotnet.microsoft.com/download/dotnet/10.0)

### 编译运行

```bash
# 克隆仓库
git clone https://github.com/<syml12138>/DelayCtrl.git
cd DelayCtrl

# 编译并运行
dotnet run
```

### 使用步骤

1. 在"连接设置"区域输入继电器设备的 **IP 地址**和**端口号**
2. 点击 **连接** 按钮建立 UDP 连接
3. 点击各路继电器的 **开启/关闭** 按钮发送控制指令
4. 观察指示灯颜色变化确认设备状态（🟢 开启 / ⚪ 关闭）
5. 底部日志区可查看完整的收发数据记录


### 核心代码结构（Form1.cs）

| 区域                   | 方法                    | 功能说明                            |
| ---------------------- | ----------------------- | ----------------------------------- |
| **连接管理**           | `Connect()`             | 建立 UDP 连接，启动异步接收循环     |
|                        | `Disconnect()`          | 断开连接，释放资源                  |
|                        | `SetRelayButtonsEnabled()` | 批量启用/禁用继电器按钮           |
| **数据接收与响应解析** | `ReceiveLoopAsync()`    | 异步接收循环，持续监听设备响应      |
|                        | `ProcessResponse()`     | 校验响应格式，提取命令字节          |
|                        | `UpdateRelayStatus()`   | 根据命令字节更新继电器界面状态      |
|                        | `SetRelayState()`       | 更新指定继电器的指示灯和文字        |
| **指令发送**           | `SendCommand()`         | 发送指令字节并记录日志              |
|                        | `AppendLog()`           | 追加带时间戳的日志消息              |
| **按钮事件**           | `BtnRelayXOn/Off_Click` | 各路继电器开关按钮点击事件          |
|                        | `BtnAllOn/Off_Click`    | 全通道开关按钮点击事件              |

## 二次开发指南

### 新增继电器通道

1. 在 `Form1.Designer.cs` 中添加新通道的 GroupBox、指示灯 Label、状态 Label、开/关 Button
2. 在 `Form1.cs` 中：
   - `SetRelayButtonsEnabled()` 中添加新按钮的启用/禁用
   - `UpdateRelayStatus()` 的 `switch` 中添加新命令字节的分支
   - `SetRelayState()` 的 `switch` 中添加新通道的控件映射
   - 添加新按钮的 `Click` 事件处理方法，调用 `SendCommand()` 发送对应指令

### 修改通信协议

- 发送指令格式修改：编辑各 `BtnRelayXOn/Off_Click` 方法中的字节数组
- 响应解析修改：编辑 `ProcessResponse()` 中的校验逻辑和 `UpdateRelayStatus()` 中的命令字节映射

### 切换为 TCP 连接

将 `UdpClient` 替换为 `TcpClient`，调整 `Connect()`、`Disconnect()`、`ReceiveLoopAsync()`、`SendCommand()` 中的收发逻辑即可。
